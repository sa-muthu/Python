# 1.00 Python Basics and Style Guide

This guide covers the fundamental concepts of executing Python code and the official style guidelines for writing maintainable code.

## 1. The REPL (Read–Eval–Print Loop)

REPL stands for **Read–Eval–Print Loop**. It’s an interactive programming environment where code is executed line by line. It is a great tool for testing small snippets of Python code.

Here’s what each part means:
- **Read:** The system reads your input (a command or expression).
- **Eval (Evaluate):** It executes that input.
- **Print:** It shows the result.
- **Loop:** It repeats this process continuously.

**Example of REPL in action:**
```python
>>> 2 + 2  # [reads and evaluates]
4          # [prints]
>>> print("hello")  # [loops back]
hello
```

## 2. How to Run a Python Script from the Terminal

To run a `.py` file from the terminal, follow these straightforward steps:

1. **Navigate to the file’s directory**

   Use the `cd` (change directory) command to go to the folder containing your script:
   ```bash
   cd /path/to/your/file
   ```

2. **Run the script**

   Execute the file using the `python` command followed by the filename:
   ```bash
   python your_script.py
   ```
   *(Note: If your system uses Python 3 explicitly, use `python3 your_script.py` instead.)*

   **Example:**
   ```bash
   cd ~/Documents
   python3 test.py
   ```

## 3. PEP 8: The Official Style Guide for Python

PEP 8 is the official style guide for Python. It defines how Python code should be written so that it is consistent, readable, and maintainable. Python follows the spirit of "readability counts" (from *The Zen of Python*).

### Why PEP 8 Matters
Python code is often read more than it is written. PEP 8 ensures that:
- Code is easy to understand by others (and your future self).
- Teams can collaborate without style conflicts.
- Bugs are easier to spot.

### Key PEP 8 Guidelines

#### 1. Naming Conventions (Very Important)
- **Variables & Functions:** `snake_case`
  ```python
  total_score = 100
  def calculate_average():
      pass
  ```
- **Classes:** `PascalCase`
  ```python
  class StudentRecord:
      pass
  ```
- **Constants:** `UPPER_CASE`
  ```python
  MAX_LIMIT = 50
  ```

#### 2. Indentation and Spacing
- Use **4 spaces** per indentation level (do not use tabs).
  ```python
  def func():
      if True:
          print("Correct")
  ```
- Add spaces around operators for better readability.
  ```python
  x = 5 + 3   # good
  x=5+3       # bad
  ```

#### 3. Line Length
- Keep lines to **79 characters or fewer**.
- Break long lines up appropriately:
  ```python
  result = some_function(arg1, arg2,
                         arg3, arg4)
  ```

#### 4. Imports
- Put imports at the very top of the file.
- Group them properly (standard library, third-party, local).
  ```python
  import os           # standard library
  import numpy as np  # third-party
  
  from mymodule import my_function  # local
  ```

#### 5. Blank Lines
- Use blank lines to logically separate code sections and functions:
  ```python
  def func1():
      pass
  
  
  def func2():
      pass
  ```

#### 6. Comments
- Write clear, useful comments that explain *why* something is done, rather than *what* is done.
  ```python
  # Calculate average score
  avg = total / count
  ```
- Avoid obvious comments:
  ```python
  x = x + 1  # increment x (unnecessary)
  ```

#### 7. Docstrings
- Use docstrings to describe what functions or classes do.
  ```python
  def add(a, b):
      """Return the sum of a and b."""
      return a + b
  ```

#### 8. Comparisons and Booleans
- Compare singletons like `None` using `is` or `is not`, never `==`.
  ```python
  if x is None:      # good
  if x == None:      # bad
  ```
- For empty sequences/collections (strings, lists, etc.), use their implicit boolean value:
  ```python
  if not items:      # good
  if len(items) == 0:  # less preferred
  ```

* Is it readable?
* Are names clear?
* Is spacing clean?
